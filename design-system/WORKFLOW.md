# AIO 작업 표준 — 3-SET 워크플로우

> 모든 페이지·섹션 작업은 아래 3-SET을 **한 번도 빠뜨리지 않고** 적용한다.
> 이 문서는 AI(Claude 등) 가 작업 시작 시 첫 단계에서 반드시 따른다.

---

## 🎯 3-SET (모든 작업의 시작점)

### SET 1 — 디자인 스킬 (Design Knowledge)
- **`ui-ux-pro-max`** skill 호출 → 업종별 디자인 시스템·컬러·폰트·안티패턴 확인
- **`frontend-design`** skill 호출 → 컴포넌트 디테일·anti-AI-aesthetic 패턴
- 결과: 현재 페이지·섹션의 톤·스타일·컬러 의도 확인

### SET 2 — 카피라이터 모드 (Copywriter Lens)
모든 텍스트를 다음 기준으로 검수:

- **한글 자연스러움**: 어색한 외래어·번역체 제거
- **용어 통일**: 같은 개념은 한 단어로 (예: 카테고리/분야 혼용 X)
- **강조 위치**: 의뢰 결정에 영향 주는 핵심 명사·숫자만 `<strong>` 처리
- **줄바꿈 의도**: 모든 `<br />` 위치는 명시적 의도가 있어야 함
- **말투 일관성**: 다·습니다 vs 합니다 등 페이지 내 한 결로
- **영문/한글 분리**: 영문 디스플레이는 매거진 무드용, 한글 본문은 Pretendard

### SET 3 — 디자이너 모드 (Designer Lens)
모든 시각 요소를 토큰 기반으로 검수:

- **letter-spacing**: 디스플레이 `-0.014em`, 헤드 `-0.01em`, 본문 `0`
- **line-height**: 한글 본문 `1.8`, 영문 디스플레이 `1.0`, 캡션 `1.5`
- **섹션 상하**: `var(--space-section)` (모바일 64 → PC 120)
- **좌우 가장자리**: `var(--space-edge)` (모바일 20 → PC 40)
- **블록 간격**: `var(--space-block)` (모바일 32 → PC 56)
- **본문 폭**: `max-w-[42ch]` ~ `max-w-[60ch]` (글자수 기준)
- **정렬**: 매거진 톤 기본 `text-align: center`. 좌정렬은 의도 있을 때만
- **인라인 `clamp()` 금지**: 항상 `var(--text-*)` 토큰 사용

## 📋 페이지 작업 표준 순서

1. **현재 페이지 가이드 읽기** — `design-system/pages/{슬러그}.md`
2. **SET 1 호출** — 디자인 스킬에서 가이드 받기
3. **SET 2 적용** — 카피 전수 검수
4. **SET 3 적용** — 토큰화 + 정렬·간격 디테일
5. **모바일/태블릿/PC 4단계 검증** (375/768/1024/1440)
6. **안티패턴 체크리스트** — MASTER.md §10 통과
7. **TypeScript + 빌드 검증** — `npx tsc --noEmit` 통과
8. **커밋 + 푸시**

---

## 🚫 금지 항목

- 인라인 `clamp(N, vw, M)` 사용 (토큰으로만 대체)
- 매직 넘버 (`fontSize: 24` 같은 인라인 숫자)
- 자연 wrap 의존 (모든 줄바꿈은 `<br />` 명시)
- 한글 영문 혼용 시 띄어쓰기/letter-spacing 미조정
- 컬러 하드코딩 (`#1A1614` 같은 직접 hex; 토큰만)
