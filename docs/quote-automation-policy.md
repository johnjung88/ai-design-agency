## 기본 운영 원칙

1. 2026-05-31까지 접수 건은 `이벤트가`를 기본 판매가로 사용한다.
2. 2026-06-01부터는 `정상가`를 기본 판매가로 사용한다.
3. 숨고, 크몽, 이메일 직접 문의 가격은 동일하게 둔다.
4. 작업 범위, 디자인 수준, 일정, 운영 포함 여부를 필수 판단 축으로 둔다.
5. 기본 견적은 `최소가`로 안내하고, 자료 상태나 기능 범위가 불명확하면 `검토 후 확정`으로 표시한다.
6. 범위 밖 추가 비용은 작업 전 먼저 안내한다.
7. 납품 후 14일 동안 계약 범위 내 기본 A/S를 제공한다.
8. 안내한 납기가 지연되면 10% 보상 할인을 적용한다.
9. 재의뢰 고객은 -20% 할인을 유지한다. 단, 급행비와 외부 실비에는 자동 적용하지 않는다.
10. 원본 파일은 공통 무료 제공으로 본다. PSD, AI, Figma, Source Code 등은 기본 제공 가능 항목이며 저작권 양도 포함으로 관리한다.
11. 5일 내 완성 보장은 기본 원칙이지만 풀커스텀, 자동화 PREMIUM, 영상 PREMIUM은 예외 가능 항목으로 표시한다.
12. 세금계산서는 사업자 인증 완료 전에는 `인증 완료 후 발행 가능`, 인증 완료 후에는 `발행 가능`으로 안내한다.

## 견적 산정 순서

1. `category`와 `subtype`으로 서비스와 후보 패키지를 고른다.
2. `description`에서 수량 단서를 추출한다. 예: `5페이지`, `20장`, `3개`, `영문`, `API`, `DB`, `카페24`.
3. 후보 패키지의 기본가를 선택한다.
4. 고정 추가 옵션을 더한다. 예: 페이지 추가, 언어 추가, 슬라이드 추가, 비율 추가.
5. 급행이면 기본가와 고정 추가 옵션의 합계에 급행율을 적용한다.
6. 재의뢰 할인 또는 보상 할인이 명시된 경우 마지막 단계에서 적용한다.
7. 불확실한 범위가 있으면 최종 금액 대신 `예상 범위`와 `확정에 필요한 질문`을 함께 낸다.

계산식:

```text
subtotal = base_price + fixed_addons
rush_fee = rush ? round(subtotal * rush_rate) : 0
discount = eligible_discount ? round((subtotal + rush_fee) * discount_rate) : 0
estimate_total = subtotal + rush_fee - discount
```

권장 기본값:

| 항목 | 기본값 |
| --- | ---: |
| 일반 급행 | +30% |
| 복잡한 급행 | +50% |
| 당일 고난도 급행 | +70% |
| 재의뢰 할인 | -20% |
| 납기 지연 보상 | -10% |

| 상세페이지 | 제품 사진 보유 여부, 목표 플랫폼, 원하는 길이/섹션 수, 촬영 필요 여부 |
| PPT 디자인 | 현재 원고/PPT 보유 여부, 장수, 발표 목적, 리서치 필요 여부 |
| 자동화·앱 | 입력 데이터, 처리 규칙, 출력 형태, 반복 주기, API/DB 계정 보유 여부 |
| 영상 콘텐츠 | 영상 길이, 비율, 보유 소스, 자막/BGM, 촬영/성우 필요 여부 |

## 예시 계산

### 예시 1: 웹사이트 5P + 영문 1언어 + 급행

```text
base_price = 300,000
fixed_addons = 20,000
rush_fee = (300,000 + 20,000) * 30% = 96,000
estimate_total = 416,000
```

고객 안내: `홈페이지 기본 5P + 영문 1언어 + 급행 기준 416,000원부터`

### 예시 2: PPT 24P

```text
base_price = 100,000
extra_slides = 4 * 5,000 = 20,000
estimate_total = 120,000
```

고객 안내: `PPT 20P 패키지 + 추가 4장 기준 120,000원부터`

### 예시 3: 자동화 단순 + 소스 제공

```text
base_price = 100,000
source_delivery = 0
estimate_total = 100,000
```

고객 안내: `단일 업무 자동화 + 소스 제공 기준 100,000원부터`

### 예시 4: 앱 MVP + 화면 2개 추가 + API 1개 추가

```text
base_price = 1,500,000
extra_screens = 2 * 50,000 = 100,000
api_integration = 100,000
estimate_total = 1,700,000
```

고객 안내: `고급 자동화/앱 MVP + 화면 2개 추가 + API 1개 추가 기준 1,700,000원부터`

## 현재 불일치 및 정리 필요 항목

| 항목 | 현재 상태 | 권장 처리 |
| --- | --- | --- |
| 로고·명함 가격 | `AIO_가격관리.xlsx`와 `docs/marketplace/*`는 10,000/30,000/100,000원, `lib/services-data.ts`는 29,000/49,000/99,000원 | 자동 견적은 엑셀 기준인 10,000/30,000/100,000원 사용. 사이트 노출가 수정 필요 |
| 로고·명함 패키지명 | 사이트는 `입문형 로고·명함`, `실속형 로고 3안·명함`, `창업형 브랜드 키트`; 플랫폼 문서는 `로고 단독`, `로고+명함`, `풀브랜딩` | 고객 응답은 플랫폼형 패키지명으로 단순화 |
| 번들 가격 | `components/sections/bundle-cards.tsx`에 별도 번들가가 있으나 플랫폼 가격표에는 없음 | 자동 견적에서는 번들을 별도 상품으로만 사용하고 기본 패키지 계산에는 섞지 않음 |
| 수정 횟수 | 일부 사이트 데이터와 플랫폼 문서의 수정 횟수가 다름 | 플랫폼 원고의 STANDARD/DELUXE/PREMIUM 수정 횟수를 우선 |
| 원본 파일 제공 | 기존 문서 일부는 원본/소스 제공을 유료 옵션으로 둠. 엑셀은 공통 무료 제공으로 확정 | 자동 견적에서는 무료 처리 |
| 상세페이지 수정 | 단가표는 패키지별 1~2회, 추가요금기준은 기본 상세 1장에 수정 무제한으로 표기 | 자동 견적에서는 `범위 내 수정 제공, 대규모 변경은 +10,000원`으로 안내 |
| 외부 실비 | 도메인, 호스팅, 유료 이미지/폰트/음원, API 사용료가 가격표에 없음 | 견적서에는 `외부 실비 별도`로 고정 표기 |

## 자동화 구현 메모

다음 단계에서 코드화한다면 `lib/services-data.ts`의 문자열 가격과 문서 가격을 직접 파싱하지 말고, 별도 구조화 데이터로 분리하는 편이 좋다.

권장 파일:

- `lib/quote-pricing-policy.ts`: 숫자 기준 가격, 패키지, 추가 옵션, 할인율
- `lib/quote-estimator.ts`: 입력값을 받아 견적 결과와 확인 질문을 반환
- `app/api/quote/route.ts`: 저장 전에 `quote_estimate` 필드를 만들어 `draft_response`에 자동 초안 저장

권장 결과 타입:

```ts
type QuoteEstimate = {
  category: string;
  packageName: string;
  basePrice: number;
  addons: { name: string; amount: number }[];
  rushFee: number;
  discounts: { name: string; amount: number }[];
  totalMin: number;
  timeline: string;
  confidence: "high" | "medium" | "low";
  questions: string[];
  responseDraft: string;
};
```
