## 🚀 배포 5단계

### 1. Supabase 마이그레이션 적용

Supabase Dashboard → SQL Editor 또는 CLI:

```bash
# CLI 방식
supabase db push
# 또는 Dashboard에서 supabase/migrations/004_expenses.sql 직접 실행
```

확인:
```sql
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' AND table_name IN ('expenses','recurring_expenses');
SELECT count(*) FROM recurring_expenses;  -- 4건 (시드 데이터)
```

### 2. 환경 변수 추가 (Vercel)

Vercel Dashboard → AIO 프로젝트 → Settings → Environment Variables:

| 키 | 값 | 환경 |
|---|---|---|
| `CRON_SECRET` | 32자 이상 랜덤 문자열 | Production, Preview, Development |

생성 방법 (PowerShell):
```powershell
# 32바이트 랜덤 = 64자 hex
[Convert]::ToHexString([System.Security.Cryptography.RandomNumberGenerator]::GetBytes(32)).ToLower()
```

또는 Node:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 3. 코드 푸시 + Vercel 배포

```bash
cd "C:\Users\PC\OneDrive\Desktop\솔로프리너\AIO_SITE"
git add lib/admin/telegram.ts lib/admin/telegram-templates.ts lib/admin/briefing.ts app/api/cron vercel.json supabase/migrations/004_expenses.sql docs/_aio_bot_specs
git commit -m "feat(telegram): unified message templates + daily/weekly cron"
git push origin main
```

Vercel이 자동 배포. 약 1~2분.

### 4. 수동 테스트 (배포 후 즉시)

```powershell
# CRON_SECRET 값 (Vercel에 설정한 것과 동일)
$secret = "여기에_시크릿"
$base = "https://aio-make.com"

# Daily 테스트
curl.exe -X GET "$base/api/cron/daily" -H "Authorization: Bearer $secret"

# Weekly 테스트
curl.exe -X GET "$base/api/cron/weekly" -H "Authorization: Bearer $secret"
```

성공 시 텔레그램에 메시지 도착. 응답 JSON: `{ "ok": true, "route": "daily", "at": "..." }`

⚠️ 인증 실패(401): `CRON_SECRET` 값 확인.
⚠️ 503: Vercel에 `CRON_SECRET` 환경변수 미설정.

### 5. Vercel Cron 활성 확인

Vercel Dashboard → AIO 프로젝트 → Cron Jobs 메뉴:
- `/api/cron/daily` — `0 22 * * *` (매일 UTC 22:00 = KST 07:00 다음날)
- `/api/cron/weekly` — `0 22 * * 0` (일요일 UTC 22:00 = 월요일 KST 07:00)

다음 실행 시각이 표시되는지 확인.
