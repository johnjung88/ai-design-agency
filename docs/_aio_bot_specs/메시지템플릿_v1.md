## 1. 알림 카테고리 (3종)

| 카테고리 | 트리거 | 발송 시점 |
|---|---|---|
| **즉시 알림 (Event)** | 사이트 이벤트 발생 시 | 실시간 |
| **일일 브리핑 (Daily)** | Vercel Cron | 매일 KST 07:00 |
| **주간 요약 (Weekly)** | Vercel Cron | 매주 월 KST 07:00 |

→ 월요일 KST 07:00 = *2개 메시지* (일일 + 주간)

## 2. 통일 형식 규칙

### 2.1 날짜·시간

| 표기 | 형식 | 예 |
|---|---|---|
| 날짜 | `M/D (요일)` | `5/8 (금)` |
| 시간 | `HH:MM` | `09:00` |
| 날짜+시간 | `M/D (요일) HH:MM` | `5/8 (금) 09:00` |
| 마감 | `D-N` 또는 `오늘 마감` | `D-1`, `오늘 마감` |
| 기간 | `M/D ~ M/D` | `5/12 ~ 5/18` |

### 2.2 금액

| 표기 | 형식 |
|---|---|
| 정확 | `*₩550,000*` (마크다운 굵게) |
| 만원 단위 | `*55만*` |

### 2.3 마크다운 (Telegram parse_mode: Markdown)

```
*굵게* — 핵심 정보·금액·이름
_기울임_ — 부가 설명·메타
`코드` — 명령어·ID·플랫폼명
```

→ `sendTelegramMessage` 호출 시 `parse_mode: "Markdown"` 옵션 활성

### 2.4 섹션 구분

긴 메시지(일일/주간)에서만:

```
━━━━━━━━━━━━━━━━━━━━━
```

### 2.5 이모지 표준

| 이모지 | 의미 |
|---|---|
| 🟢 | 정상·운영 중 |
| 🟡 | 진행 중·대기 |
| 🔴 | 긴급·문제 |
| ✅ | 완료 |
| ❌ | 실패·먹튀 |
| ⚠️ | 주의 |
| 💰 | 매출 |
| 💸 | 지출 |
| 💎 | 손익 |
| 📅 | 일정 |
| 🎯 | 목표 |
| 📥 | 신규 유입 |
| 📊 | 통계·요약 |
| 🤖 | 챗봇 |
| 💡 | 인사이트 |
| 👉 | 다음 액션 |
| 🔥 | P0 (필수) |

→ 메시지당 카테고리 이모지 1~2개 + 항목 이모지

## 3. 메시지 템플릿

### A. 즉시 알림 (Event)

```
{이모지} *{제목}*

{핵심 정보 1줄}
{메타: 금액·날짜·고객 등 2~4줄}

👉 _{다음 액션 1줄}_
```

**예시:**
```
💰 *빌더스 PPT 입금 완료*

`5/2` 작업 → `5/7` 입금
금액: *₩100,000*
입금 누계: *₩200,000* _(4건)_

👉 _후기 요청 추천 (5점 강요 X)_
```

## 4. 데이터 소스 (Supabase)

| 데이터 | 테이블 | 비고 |
|---|---|---|
| 진행 중 외주 (오늘 할 일) | `projects` (status `in_progress`, due_date 기준) | 정렬: due_date 빠른 순 |
| 매출 확정 | `projects` (월 단위 created_at) | sum(contracted_amount) |
| 입금 완료 | `invoices` (paid_amount > 0, payment_status `paid`) | 월 누계 |
| 입금 대기 | `invoices` (status `unpaid`/`partial`) | outstanding_amount |
| 지출 | `expenses` (월 단위 date) | 신규 테이블 (마이그레이션 004) |
| 신규 견적 | `inquiries` | 24시간 / 주간 |
| 챗봇 대화 | `chatbot_logs` | 24시간 / 주간 |

## 5. Vercel Cron 스케줄

`vercel.json`:

```json
{
  "crons": [
    { "path": "/api/cron/daily", "schedule": "0 22 * * *" },
    { "path": "/api/cron/weekly", "schedule": "0 22 * * 0" }
  ]
}
```

- `0 22 * * *` = UTC 22:00 = KST 07:00 (다음날)
- `0 22 * * 0` = 일요일 UTC 22:00 = 월요일 KST 07:00

**보안**: Vercel cron 호출에는 `Authorization: Bearer ${CRON_SECRET}` 헤더 자동 포함. route handler에서 검증.
